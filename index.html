--- title: Original Civilization Board Game Utility layout: markdown isCurrent: true appurl: http://app2.civbuddy.org --- ## Welcome to {{ site.name }}. {{ site.name }} is designed to support players of Francis Tresham's [original Civilization board game](https://boardgamegeek.com/boardgame/71/civilization) by helping with the tactical planning when buying civilization cards in game phase 10. This is not a computer game, but a helper application for the board game. You need the board game for {{ site.name }} to be useful to you. In later stages of the game, the game rules lead to phases of long-winded calculations in the upper three-digit numbers. With {{ site.name }}, this can be solved quickly and easily, so that everyone can concentrate on all the fun parts of the game. On Android, the process is the same as on iOS, but the *Add to Home Screen* option is in the drop-down menu of the Chrome browser. ## Development Setup This website is primarily intended to centralize development related information. ### Setting up a local development workspace In order to build {{ site.name }} yourself, follow these steps: 1. Make sure you have the prerequisites set up: - [Git](http://git-scm.com/) - [NodeJs and NPM](https://nodejs.org/en/download/) - The download for Node includes NPM. 1. NodeJs and NPM should be on the PATH. 1. Clone the {{ site.name }} repo by running on the command line: git clone https://github.com/{{ site.github }}.git 1. `cd civbuddy` 1. `npm install` 1. `npm run build`\\ This will create a {{ site.name }} distributable in *build/dist* that can be used for development. 1. In order to create a version for production, run `npm run hash` in order to create hashed files for cache handling and remove source maps. The {{ site.name }} code as found on GitHub is IDE agnostic, so you may use your favorite IDE to edit the sources. We use Visual Studio Code, which is why there are some settings for Visual Studio Code. They are not critical to {{ site.name }} development.
